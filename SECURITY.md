# Security Policy

## Supported Versions

Currently, we only provide security updates for the latest stable release.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do NOT open public issues for security vulnerabilities.**

If you discover a potential security vulnerability in Sunder, please notify us by sending an email to [security@sunder.app].

### Our Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
2. **Investigation**: We will investigate the issue and determine its severity.
3. **Fix**: We aim to provide a fix or mitigation within a reasonable timeframe (typically 7-14 days for critical issues).
4. **Recognition**: We will provide attribution in our release notes for reporters of valid security issues, unless anonymity is requested.

## Coordinated Disclosure

We follow coordinated disclosure practices. We ask that you do not share details of the vulnerability until a fix has been released and we have agreed on a disclosure timeline.
